# Local-First Knowledge Bases

**Maya Brennan:** This is Signals & Noise, I'm Maya Brennan. My guest today is Priya Nair, who has spent a decade building knowledge tools — first inside a SaaS company, now very deliberately outside one.

**Priya Nair:** Deliberately is the word. I helped build a beautiful cloud knowledge base that forty thousand teams used, and I watched what happened when we deprecated the export feature for a quarter. People's notes were hostages.

**Maya Brennan:** Hostages with a subscription fee.

**Priya Nair:** Exactly. So my rule now is simple: if my notes can't survive the company that made the editor, they were never my notes.

**Maya Brennan:** Which lands you on plain markdown files.

**Priya Nair:** Plain markdown in a folder, yes. It sounds like a regression and it's actually the most advanced position. Markdown is boring in the way load-bearing things should be boring.

**Maya Brennan:** The counterargument is that SaaS tools give you sync, search, collaboration, all the toys.

**Priya Nair:** And for a team wiki, take the toys! But a personal knowledge base has one user and a fifty-year time horizon. No vendor roadmap survives fifty years. A folder of text files has already survived longer than most of the companies in this space.

**Maya Brennan:** You also make an argument about agents that I want you to repeat on the record.

**Priya Nair:** The one that gets me in trouble. Fine: the killer feature of plain text is that a language model can be a full citizen of your wiki — it can read every page, write every page, and never needs an API key to your life.

**Maya Brennan:** Because the interface is just the filesystem.

**Priya Nair:** Right. Every SaaS tool is now bolting on an AI sidebar that sees one document at a time. Meanwhile a local agent with grep can see the whole graph. The architecture nobody is selling is better than the one everybody is selling.

**Maya Brennan:** What breaks? Be honest. Merge conflicts? Mobile?

**Priya Nair:** Mobile is genuinely worse, I won't pretend otherwise. And links rot if you rename files carelessly. But those are weekend problems. Vendor lock-in is a decade problem.

**Maya Brennan:** Weekend problems versus decade problems — I'm stealing that.

**Priya Nair:** It's a markdown file, Maya. You can't steal what's already portable.

**Maya Brennan:** Priya Nair, everyone. Her essays are at her own domain, naturally, in plain HTML.

**Priya Nair:** Hand-written HTML. Thanks for having me.
